<h1><b><u> Object Oriented Programming in python</u></b></h1>
Object-Oriented Programming (OOP) in Python is a way of structuring code using classes and objects so that programs become easier to understand, reuse, and scale. Think of it as modeling real-world things (like a Car or Student) inside your code, where each object has data (attributes) and actions (methods).

<h2><u>Class</u></h2>
A class is like a blueprint or design plan. It doesn’t exist physically but tells us what an object should look like and what it can do. Imagine the blueprint of a house: it defines rooms, doors, and windows, but it’s not a real house yet. In OOP, a class defines attributes (like name, age) and behaviors (like speak, walk). For example, a “Car” class might describe that every car has wheels, an engine, and can drive. The class itself is just the idea; it’s only when we build a car from that blueprint that we get something real to use.

<h2><u>Object</u></h2>
An object is the real-world thing created from a class. If a class is the blueprint of a car, then your red Maruti or my blue Honda are actual objects. Each object has its own unique values but follows the same design. For example, two students in a school are both objects of the “Student” class. One may have the name Siddharth and roll number 101, while another has Ananya and roll number 102. Both are students but with different details. Objects make the abstract blueprint useful, because they are the actual entities we interact with in a program.

<h2><u>Encapsulation</u></h2>
Encapsulation means keeping related data and actions together inside one unit. Think of a mobile phone: it has a battery, screen, and apps, but you don’t see the wires or circuits inside. Everything is neatly packed, and you only interact with the phone as a whole. In OOP, encapsulation ensures that attributes (like a student’s name) and methods (like introduce) are bundled inside the class. This protects data from being misused and makes the program organized. Just like you don’t open your phone to charge the battery directly, you use the charging port — encapsulation hides complexity and provides controlled access.

<h2><u>Inheritance</u></h2>
Inheritance is about passing features from one class to another. It’s like family traits: a child inherits qualities from parents but can also have unique abilities. In OOP, if we have a “Person” class with features like eat and sleep, a “Student” class can inherit these features automatically. The student doesn’t need to redefine eating or sleeping; they come from the parent class. But the student can add new features like study or play. This saves time, avoids repetition, and makes code reusable. Just like every dog is an animal, every student is a person — inheritance connects them.

<h2><u>Polymorphism</u></h2>
Polymorphism means same action, different behavior. Imagine asking different animals to “speak.” A dog barks, a cat meows, and a cow moos. The action “speak” is the same, but the result depends on the animal. In OOP, polymorphism allows different classes to define the same method but behave differently. This makes programs flexible and adaptable. For example, in a school system, both teachers and students might have an “introduce” method. A teacher says, “I teach Math,” while a student says, “I study Science.” The action is identical, but the outcome varies depending on the object performing it.
